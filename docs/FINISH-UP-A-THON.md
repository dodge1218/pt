## Origin Story

ProofTicket started as a more formal way to collaborate on a project with a friend.

That collaboration did not last. The disagreement on direction ended the original version, but the underlying workflow problem remained: if humans and agents are both doing real work, where does that work live once it needs review, approval, evidence, or follow-up?

This submission is the cleaned-up surviving slice of that idea.

## What The Project Shows

ProofTicket shows a real workflow:

1. an agent proposes work,
2. a human approves or rejects it,
3. evidence stays attached,
4. later reviewers do not need to replay the whole chat or terminal.

That is the core promise. The project does not need to pretend every future product surface is already built.
