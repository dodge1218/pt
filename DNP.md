# DO NOT PUBLISH (DNP) Protocol

## Scope
Applies to ALL public-facing code, docs, READMEs, landing pages, commit messages, API docs, blog posts, tweets, and demo content.

## NEVER publish or reference publicly:
- The chronos/kairos framework, tier system, or classification vocabulary
- Terms: chronoloop, chronohoe, kairotic, kairos initiate, timework, BioKairos
- The M1/G2/Sub-M1 classification system
- ICE/PIE heuristic names (DRF, TIE, NOV, SAL, DEP, VCD, SDS)
- The looper concept, looper tiers, population-scale timework
- Any framework connecting consumer spending patterns to loop behavior
- Ryan's personal chronos/kairos arc or self-analysis
- The Chronokid show concept, character, tweets
- Pseudonym lists (Quiet Rollcall)
- The OPSEC protocol itself
- Any content from the ChatGPT export corpus that reveals the framework

## What IS okay publicly:
- "Kairos" as a product name (common Greek word, no framework context)
- "Right moment delivery" as a feature description (generic enough)
- "Thinking profile" and "passion matching" (original to the product)
- "Voice handshake" (original to the product)
- Standard dev/startup language

## Rule:
If you're unsure whether something reveals the framework → don't publish it.
The framework is the moat. The product is the surface. They never touch in public.

## Internal docs:
Framework details live ONLY in:
- Local workspace files (never committed to public repos)
- Private memory files
- The ChatGPT export corpus
- Conversation with Ryan directly
